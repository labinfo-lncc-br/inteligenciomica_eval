Categoria A — float("NAN") ↔ float("nan") (6 mutantes)
Mutantes: x__iqr__mutmut_5, x__critical_failure_rate__mutmut_6,
          x__win_rates__mutmut_8, aggregate_all__mutmut_21,
          rank_score.compute__mutmut_11, final_score.compute__mutmut_22

Prova: A função Python built-in float() aceita "nan", "NAN", "Nan" e qualquer
variação de capitalização de forma case-insensitive (CPython Objects/floatobject.c,
função case_insensitive_match). Em qualquer versão suportada do CPython (3.11+),
float("NAN") is math.isnan() → True, idêntico a float("nan"). Nenhuma asserção
de teste baseada em math.isnan(), comparação aritmética ou serialização pode
distinguir os dois. Observabilidade externa idêntica para toda entrada válida.

Categoria B — n=4 é o default de statistics.quantiles (1 mutante)
Mutante: x__iqr__mutmut_11

Prova: A assinatura de statistics.quantiles (CPython Lib/statistics.py) é
  def quantiles(data, *, n=4, method='exclusive')
O parâmetro n=4 corresponde a quartis — exatamente o valor omitido pelo mutante.
Portanto statistics.quantiles(values, method="inclusive") e
statistics.quantiles(values, n=4, method="inclusive") são chamadas idênticas para
toda entrada, em toda versão do Python 3.11+. O resultado é bit-a-bit idêntico.

Categoria F — Guarda NaN com or→and em rank_score.compute (3 mutantes)
Mutantes: compute__mutmut_1, compute__mutmut_2, compute__mutmut_3

Prova (IEEE 754 NaN propagation): A guarda explícita em rank_score.compute:
  if isnan(median_score) or isnan(failure_rate) or isnan(win_rate) or isnan(cfr):
      return RankScore(float("nan"))
é uma otimização de curto-circuito; a aritmética subsequente propaga NaN
igualmente sem ela. Formalmente:
  RankScore = w_m*median + w_f*(1-failure) + w_w*win - w_p*cfr
Para qualquer campo xi = NaN com peso wi > 0 (e todos os pesos são > 0 por
validação do construtor), wi*NaN = NaN (IEEE 754, §6.2), e NaN + qualquer_finito
= NaN. Logo, sum = NaN e RankScore(NaN) é retornado.

Os mutantes trocam `or` por `and` em pares adjacentes:
- mutmut_1: o ramo `or isnan(win_rate) or isnan(cfr)` vira
             `or isnan(win_rate) and isnan(cfr)`. Se apenas win_rate=NaN:
  original: guarda dispara → RankScore(NaN)  ✓
  mutante:  guarda não dispara → w_w*NaN=NaN na soma → RankScore(NaN)  ✓  (idêntico)
- mutmut_2/3: raciocínio análogo para os outros pares.

Para TODA entrada onde qualquer campo é NaN, a saída observável é RankScore(NaN)
tanto no original quanto no mutante. Nenhuma entrada válida distingue os mutantes.
Nota: estes mutantes usam o operador booleano `and`/`or`, que não consta da lista
de operadores aritméticos proibidos do Prompt B item 3 (+, -, *, /, <, >, <=, >=).
